# VibeCoding 技巧之UI控制术和飞书消息通知

大家好，我是唐斩，这是第71天第87篇分享。

## 1、VibeCoding实践(一)-UI控制术👀
VibeCoding时，如何让语言控制UI生成的UI符合要求。
介绍不同阶段的两种方法
- MVP阶段，从0.1到1，用“交互-功能-视觉”描述法
- 精细化迭代阶段，从1到10，用“VibeDesign” 生成法

### MVP阶段的 “布局-功能-视觉”描述法
“布局-功能-视觉”描述法是一个提示词框架，我在VibeCoding做项目时，会在项目根目录下创建 docs文件夹，存放项目的所有文档，并创建 prd.md 描述我的需求。
需求结构是 产品定位(价值描述，解决什么问题)、产品设计(产品层面的描述，怎么解决问题)、技术设计(怎么实现产品，技术栈要求、技术架构要求等)，足够AI从0到1生成一个项目了。
其中产品设计章节就写 交互-功能-视觉 三部分内容
- 交互，描述布局和操作
  - 布局，金字塔结构，总的布局样式(比如上左右三栏布局，上中下横式三列布局)，每个区域的作用是什么(例如上部menu，左部分类列表，右部分类下数据详情)，然后迭代去描述每个区域内部的布局划分(迭代到最小不可分区域)，最后去描述区域内有哪些组件以及组件间的布局关系(例如 竖向排列，提示信息、输入框、确认按钮)
- 功能，描述用户在组件上可做的操作和效果
  - 对每个区域内的每个组件，描述用户如何与之操作，以及操作后的效果(不涉及实现细节的)，比如跳转，弹窗，提示，校验等等。
- 视觉，描述纯视觉的要求
  - 视觉风格(用词描述，科技、复古、怀旧、清新等词汇)，色系要求(蓝紫、黄绿等等)，效果要求(渐变、动画等)

### 精细阶段的 “VibeDesign” 生成法
MVP阶段验证产品后，需要更精美的UI时，可以用 AI Design 平台来生成更专业的UI了。有 产品设计、应用生成、专业设计 三类平台可以使用。使用方法都是prompts 
最终都是复制生成的前端代码到项目中，再微调使用。
- 产品设计类，适合产品经理角色使用，特点是能生成可交互原型(其实也是前端代码驱动的)
  - 可用平台，国内：墨刀(个人推荐)、uxbot，国外：uizard、visily
- 应用生成类，所有的“一句话生成app平台”都可以使用，特点是直接生成成品。
  - 可用平台，国外：v0.dev、lovable、bolt.new、googleaistudio等等
- 专业设计类，适合设计感要求高的，不生成代码的就下载图片让AI参考
  - 可用平台，国外：figma(首推，共享协作很方便)，lovart(有UI生成的提示词案例，出图后给AI参考)
此外还有组件市场，如 21st ，挑选自己喜欢的组件使用(很贴心提供了复制 prompt的功能)

**TIPS** 升级UI时，记得提示词加一句"本次修改只调整UI，不涉及功能改动，不要对已有功能有任何的影响"，会极大降低改UI把功能改坏了的风险😉。


### 2、VibeCoding技巧(二)-ClaudeCode+飞书消息通知📢
VibeCoding的你，肯定遇到过，AI干活时切换出去干别的事情，1小时后切回来一看什么都没干，停在第一步，询问我查看文件的权限(白等，崩溃 Orz)。
ClaudeCode 提供了 Hooks 能力可以解决这个问题。
- ClaudeCode 的 Hooks 提供了9个事件可以添加回调钩子事件(hook)，分别是 
  - PreToolUse 和 PostToolUse，在tool工具执行前和执行后
  - Notification，ClaudeCode发送通知，比如征求权限等
  - UserPromptSubmit，用户提交提示词时
  - SessionStart 和 SessionEnd，一次会话开始和结束的时候
  - PreCompact，会话compact之前
  - SubAgentStop，子Agent结束响应时
  - Stop，Claude结束响应时
- ClaudeCode 提供了三个 hooks 的配置生效级别，分别是
  - 全局，配置在用户根目录 ~/.claude/setting.json 下
  - 项目级，配置在项目根目录 ~/${project}/.claude/setting.json 下
  - 项目本地级，配置在项目根目录 ~/${project}/.claude/setting.local.json 下，不提交到git仓库
- ClaudeCode hooks 配置例子
```
"hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/send_feishu.sh 执行完毕了。"
          }
        ]
      }
    ]
}
```
- 飞书消息通知，在飞书开放平台创建一个应用获得应用secret，然后开启消息、个人信息权限，按如下流程调试
  - 获得 access token、获得 userId、发送消息
  - 建议只用 curl，这样可以用 shell 命令完成，不需要python环境
  - 把 curl 用 shell 脚本串联起来，把脚本配置到 hooks 的 command 命令里即可。
- 同理，钉钉、企微、slack、邮件(mailgun)的自动通知都是这样的方式。可以再配置一个环境变量用来控制通知开关，这样你紧盯着屏幕时，就不会一直给你发消息了。
