# 大家好，我是唐斩，这是第80天第97篇分享。


## 1、50vibecoding技巧(7) 避免反复修改
经常 VibeCoding 的朋友一定遇到过，和AI反复沟通，反复修改，按下葫芦浮起瓢，一直改不好的情况。
解决的办法就是三句话，方案先行，生成改进，迭代重来。
具体解释一下
- 方案先行
  - 技术方案是在向AI提供足够的上下文信息，减少AI自行发挥的空间，生成结果才能稳定可靠
  - 方案里面，至少要包含 目标，产品设计，技术设计三个部分。背后的理念是，把AI当成新伙伴，帮他进入项目，那就尽量和他讲清楚相关的信息。
  - 方案是可以和AI协作完成的，我会先写一版，然后和AI沟通，让他理解并向我提问，我来澄清，最后AI补充细节，我来审核。
  - 方案要做到你能力的上限，Review到没有任何改进空间和想不到任何没说清楚的地方了，就行。
- 生成改进
  - 让AI基于方案，完成一版代码，然后我们来测试。
  - 要准备一份测试用例，用来测试和验收AI生成的效果，复杂的情况可以和AI沟通来完善测试用例。
  - 测试发现的问题可以修改，但**千万不要加功能**，忍住，千万忍住。
  - **重点**，系统优于局部原则，发现的问题可以去改进，但是一旦发现有改不完或者改不对的情况，**不要继续修改**🙅
- 迭代重来
  - 把问题抛给AI，要求它挖掘问题根因，根据挖掘的结果，补充和完善技术方案。
  - 通过迭代，增加技术方案的信息量，减少AI的发挥空间，让生成结果更可靠。
  - 方案完善后，回到第二步，再次生成

VibeCoding到最后，还是要拼软件工程和技术经验，但要避免自己上手改细节，丢失掉AI的优势。如此三步迭代，可彻底告别反复修改。(不修改，就重新生成)。

## 2、Agent开发(4) 分享几个可学习参考的github开源Agent项目
### Java 的 Agent 框架 
- langchain4j，比 spring ai 出的更早的，langchain框架的 Java版本，和 langchain 不是一个团队的。
  - https://github.com/langchain4j/langchain4j
  - 框架的一些案例，https://github.com/langchain4j/langchain4j-examples
- Spring AI，Java事实标准Spring框架出品的 AI 开发框架，对标 langchain 的能力，定位和langchain4j对等。
  - 框架本身，https://github.com/spring-projects/spring-ai
  - 框架的一些案例，https://github.com/spring-projects/spring-ai-examples
- Spring AI Alibaba，阿里在 Spring AI 之上，集成了阿里云端产品的开发框架，如果绑定阿里云的，可以直接用这个，jar包的形式引入的 Spring AI
  - 框架本身，https://github.com/alibaba/spring-ai-alibaba
  - 框架的使用案例，https://github.com/spring-ai-alibaba/examples
 
阿里基于 Spring AI Alibaba，和热门产品搭建的开源复现版本，jmanus对应manus，deepresearch其实也类似，可以参考产品架构和agent技术架构
- JManus，https://github.com/spring-ai-alibaba/JManus，
- DeepResearch,https://github.com/spring-ai-alibaba/deepresearch

### python 的 Agent 框架/产品
- langchain
  - 最广泛使用的Agent框架，必看必了解，https://github.com/langchain-ai/langchain 
  - js的版本，官方维护的，https://github.com/langchain-ai/langchainjs 
- openmanus，https://github.com/FoundationAgents/OpenManus
  - 这个很有意思，manus出来的当晚，openmanus 号称3小时就做出来了，原因是他们团队是 metagpt，本身就已经做了一款multiagent的用于开发的产品(对标的devin)。
