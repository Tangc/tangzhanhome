# 大家好，我是唐斩，这是第75天第91篇分享。

## 1、Agent开发(3) Agent会话设计
技术架构：Java + SpringBoot + Spring AI Alibaba。产品功能：根据对话内容推荐合适的商品
今天的目标是跑通主流程，以自然语言对话为交互方式，记录几个重点
- 最开始系统应该有欢迎语，给一些场景，引导用户开始对话(完全不引导用户会懵的，不知道该说什么)
- **会话**设计(conversation)
 - 会话应该由用户发出的第一句对话触发创建。
 - 会话应该有唯一标识(conversationId)。
 - 会话包含用户和系统之间的多轮对话。
 - 一个用户历史可以有多个会话，一个用户同一时间理论上可以有多个会话。如果要限制只能一个，就加锁控制。
 - 会话以系统判断用户已回复满意结束，或以距离最后回答一定时间结束，避免资源无法回收。
- 会话的技术实现
 - 多轮对话要通过 memory 记忆能力实现，Spring AI Alibaba 提供 MemoryAdivsor 扩展记忆能力。
 - 记忆要存储，存储可以用多种介质，常见的有 内存(应用进程重启后丢失，读写速度极快)，Redis(分布式场景高速缓存，可开启持久化，读写速度快)，MySQL(关系数据库，天然持久化，读写速度普通)，理论上所有的存储都可以接入，视自己的需求而定就好。
开发期间用 内存 或 Sqlite，正式上线建议使用 Redis(依赖高可用，不开持久化) 或 MySQL。


## 2、openai举办devday，ChatGPT按部就班往超级应用发展
10月7日凌晨1点，openai举办了2025的devday，发布了一系列内容，包括
- Apps in ChatGPT， 在 ChatGPT 里可以通过 @app 调用app的能力，比如 @Figma 给我设计一个devday的海报。
 - 熟悉的操作系统的味道，只有 ChatGPT 敢在自己内部去露出别的app，而现有的产品都是用“能力”的方式给聊天框强化能力，不敢明确接别的app。ChatGPT确实往前走了一步，相信其他厂商也会快速跟进的。
 - 技术层面，ChatGPT 提供了 Apps SDK 帮助开发者，从文档看，就是部署一个符合ChatGPT规范的 MCP Server ，就可以实现。但怎么审核上线，和能对多少用户开放，目前还看不出来。还是很有想象力的。
- AgentKit，OpenAI也提供了Agent搭建的工具包，包括 工作流画布、数据连接器和对话聊天UI。说“也”是因为，coze做了开源，dify、n8n等本身就是开源的。看来Agent生态，也是OpenAI向扩展的方向，都希望自己是标准。
- Codex 正式版(5月发布的预览版，发展很快)，并且发布3个feature，Slack集成(cursor老早就做过了)，Codex SDK(对标ClaudeCode刚发布的SDK，可快速接入 CodexCLI)，New admin tools(看监控用，Codex有云端模式，对admin的需求更强)
- Sora2、GPT5 Pro 等的API，很多三方应用的站点可以直接通过API升级能力了，包括用sora2生成视频，
总结一下，只有 Apps in ChatGPT 的想象空间很大，其他内容都属正常范畴。怎么能第一时间把自己app放到 chatgpt 里，又会产生什么新玩法，值得关注。
