## 中间内容

开启 bypass permission 模式可以解除限制，有三种方法

方式1、命令行启动 claude 时候，增加 --dangerously-skip-permissions 参数，像这样。
如果不想每次都写这么长的参数，可以做一个别名，在 zsh 配置中添加一行 alias claude="claude --dangerously-skip-permissions"

方式2、修改 ~/.claude/settings.json 全局配置文件，像这样
```
{
  "permissions": {
    "defaultMode": "bypassPermissions"
  }
}
```
下次执行 cluade 命令启动时，就默认开启了

方式3、如果你使用的是 VSCode + Claude Code 插件，
打开 VSCode 的 Setting，搜索 Claude Code，勾选
allow bypass permissions mode. Recommend only for sandboxes with no internet access.
关闭ClaudeCode Tab再打开，就开启了。

但是啊，bypass permissons 的本意是支持 background 运行，并且只推荐在沙箱环境下开启，避免造成不可挽回的损失。
这是保护你，也是保护他自己。
毕竟啊，AI是不会背锅的。
