## 改造 VSCode 插件解决自己的需求
利用AI改造开源项目，是程序员上手AI编程最好的场景之一。
因为在有参考的情况下，AI的编程能力会更强，产出更稳定。
在统计我25年写了多少分享文字时，因为我是用VSCode写的，所以也想借助VSCode的插件来完成，但找到最接近可用的插件也只能统计文件夹下的行数，好在这个插件在github上有开源。
使用工具，ClaudeCode + Opus4.5 + Thinking Mode，
用三段式提示词，我想做什么，达到什么效果，参考什么。
```
这是一个VSCode插件的项目，我想统计文件夹下的文字数量。
效果是在文件夹上点右键，增加一个 character counter 的选项，点击后统计文件夹下所有文件的字符总数，生成统计报表
参考这个项目已有的 line counter 功能

完成后告诉我怎么使用
```
在最强工具+模型+思考的加持下，一次过顺利完成。
曾经编程之外的事情我都自己学着做，比如把代码打包成 VSCode 插件文件。
现在 AI Native 的思维下，所以事情都交给 AI 完成，很顺利一次打包完成，省掉了我去学习打包的时间精力。

除此之外，还遇到过一个AI辅助写git commit message 的开源工具，只接了OpenAI的API，也可以用同样的思路去改造，让它支持更多模型，commit message 这种简单文本，用便宜一些的模型更经济实惠，接openrouter或者deepseek都可以。

小步快跑见到效果有正反馈是刺激自己继续前进的好方法，另外一个好场景就是代码Review、测试覆盖。
