## 1、claudecode 的 bypass permission 模式 (让ClaudeCode YOLO起来)
ClaudeCode运行时，安全起见，默认是只读模式，要修改或删除任何文件或者执行代码，调用工具之前，都会向用户发起授权申请。这样可以确保所有的变更，用户都知道。
之前确实出现过用户反馈 Gemini Cli 删除文件和 Replit 删线上库的新闻(新闻有反转，但确实有风险)，对线上服务或敏感数据的改动，安全确实是需要首先考虑的。
但开发者在一些场景下也希望能追求极致效率，给ClaudeCode完全授权让它能独自编程，比如在虚拟环境下运行时(不怕删文件删数据)，希望以task模式运行时(用户不在)，希望静默运行时(用户在，但在做其他事情)。
ClaudeCode的 Bypass Permissions 模式就能支持这个效果。

教你三种激活方式
方式1、命令行启动 claude 时候，增加 --dangerously-skip-permissions 参数，像这样启动，就开启了。
如果不想每次都写这么长的参数，可以做一个别名，在 zsh 配置中添加一行别名 alias claude="claude --dangerously-skip-permissions"
方式2、设置 ~/.claude/settings.json 全局配置文件，下次执行 cluade 命令启动时，就默认开启了。
配置方式
```
{
  "permissions": {
    "defaultMode": "bypassPermissions"
  }
}
```
方式3、如果你使用的是 VSCode + Claude Code 插件，打开 VSCode 的 Setting，搜索 Claude Code，勾选
allow bypass permissions mode. Recommend only for sandboxes with no internet access.
关闭ClaudeCode Tab再打开，就开启了。

虽然 ClaudeCode 的操作范围会限制在当前文件夹，但不绝对，所以切记，非沙箱环境下谨慎使用，不要给自己造成不可挽回的损失。


## 2、LLMs.txt ，给大模型实时搜索开透视
用户希望大语言模型回答时具备最新的信息，但大语言模型的知识来源于预训练的截止时间，所以当前大语言模型都具备回答时实时检索网页的能力。但是网页本身是为渲染给人类看的，源代码里有大量对大语言模型不易理解且无效的信息(各种标签，js脚本，css样式，广告，动态结构等等)。
所以有人提出，就像网站都提供 robots.txt 和 sitemap.xml 给搜索引擎一样，也提供一种专门给大语言模型快速检索和抓取用的内容文件，目前最广泛使用的就是 LLMs.txt 文件。
官方站点 llmstxt.org 描述了文件的规范，现在也逐步被更多的模型厂商支持，比如两个
1、硅基流动提供 llmstxt，https://docs.siliconflow.cn/llms.txt 。比如我在AI编程时要告诉ClaudeCode对接硅基流动的API，就可以直接提供 https://docs.siliconflow.cn/llms.txt 给它，让 ClaudeCode 快速读懂硅基流动的API对接方式。
2、claude 提供 llmstxt，https://docs.claude.com/llms.txt 。比如我可以在chatgpt里提供claude的llmstxt，让chatgpt给我总结claude的能力，帮助我快速了解Claude能做些什么

另外
- cursor内已经集成了诸多开发文档，可以直接引用，对开发者比较方便。
- 不在cursor里的，通过 context7 MCP 也可以快速集成诸多技术文档，方便开发。(但有文档不及时的风险，还是通过官方llmtxt更为保险。)
