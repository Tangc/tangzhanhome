核心思想就是当你决定要行动的时候，一定把任务拆分到小时级别，然后交给 AI Agent 完成。

目前看 MVP搭建 和 Vibe Coding 真的是天作之合。小任务可以避免LLM的幻觉问题，小步快跑尽早做出可视化的东西也可以给予我们正反馈。

快速做MVP的过程本身也是在一次一次积累经验，与其到处找课程学习做作业，不如玩真的自己做一次来的实在。

二则
最近 Vibe Coding 发现，即使在很小的项目里，让 AI 修改 feature 的难度也比增加 feature 的难度大很多。
因为修改 feature 可能涉及多个文件，很难一次找齐，上下文窗口增加更容易出现幻觉问题。
所以纯 Vibe Coding 的项目，可以尽量组件化，拆分得足够细致，可以每次修改只划定一个确定的小范围给AI去调整，尽量避免大更改。
另外一个最佳实践就是每次AI完成一个feature，都用git记录一个版本同时记录下这次使用的 prmpt，方便回退。即使要回退多次，后续依靠 prompt 也可以追溯回来。
