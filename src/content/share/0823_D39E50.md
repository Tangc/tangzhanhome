
我的分享内容会聚焦到AI Agent开发上，通过 build in public 的方式记录Java后端做 AI Agent 开发的思考和过程。
大家有什么想法可以评论区交流学习 ^ ^

1、AI Agent 的架构
三层架构，从上到下(个人总结)
- 业务层：传统业务后端开发的所有部分，包含端、页面、后端功能模块(用户、支付等都包含)
- 智能层：垂类Agent封装领域知识的部分，上下文工程，提示词工程都在这里实现
- 模型层：对接大语言模型的部分，可以是自建、API、三方API、路由API等方式，调用模型的failover也在这里

2、AI Agent 的开发方式
两种方式，推荐混合使用，vibecoding快速生成，古法手搓精调
- VibeCoding方式，用 CodeAgent 生成代码。 适合场景
 - 前端代码时。
 - 开发独立模块内容时。
 - 0到1的时。
 - 写不熟悉的语言时。
- 古法手搓方式，纯手写。适合场景
 - vibecoding cover不住上下文时
 - vibecoding 反复生成效果不佳时
 - 核心代码，质量要求极高时
 - 需要掌控代码细节时

3、AI Agent 的技术选型
编程语言，推荐不拘泥一种，选取自己擅长和适合的
- 业务层，绝对js，和CodeAgent最适配之一，轻量paas产品最丰富，适合快速迭代
- 智能层，核心资产，选自己最擅长的，极大可能要古法手搓的部分。自己主导，vibecoding配合
- 模型层，如果有模型训练，py是绝对首选。对接模型的话，主流语言都可以胜任

4、AI Agent 的框架选型
用框架的话，使用最多的是 langchain 家族。支持python和js语言。
用Java的话，有 langchain4j，基于 langchain 的 Java版本。有 Spring AI，spring出品。
国内用 Java的话，有 Spring AI Alibaba，阿里维护的基于 Spring AI 的框架。更推荐这个。
因为有国内的开源社群可以参与，问题支持和解决会更快一些。

5、怎么开始？
以做代练是最好的学习方式。可以从模仿开始，比如复现一个你用的最多的 Agent (like manus?)
也可以做一个新的 Agent 产品，不追求巅峰，但应该能解决一个实际问题。
提供两个我的思路，至少你自己是有需求的：
- 创造型，你用大模型在创造什么，就做这个方向的 Agent。比如，做图、做视频、做表格、写文章等
- 信息型，你用大模型在吸收什么，就做这个方向的 Agent。比如，看新闻、学技术、看文章等
