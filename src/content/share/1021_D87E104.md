## 1、一文讲懂 Claude Agent Skills 的意义！

### 基础介绍
10月17日，Cluade官方发文介绍 Agent Skills，并且实装上线到 Cluade 客户端，Claude Code，Claude API等场景里。
官方介绍，Agent Skills 是把专业知识，打包成可发现可使用的能力。每个能力通过一个 SKILL.md 的文件进行描述，通过 Claude 的产品在运行时主动发现、装载和使用。(本文后续都以 Claude Code 为例)

看一个官方的例子，官方在github上放了12个技能的例子，其中一个是 MCP Builder，我们从它的 SKILL.md 文件看一下怎么定义技能
- meta信息
  - name，名称，让ClaudeCode进程唯一标识技能的字段，项目内不要重复。
  - description，描述，meta信息里最重要的字段，ClaudeCode肯定是通过LLM自然语言理解使用技能的时机，用的就是 description 字段。所以官方也特别强调了 description 字段要写得简练、清晰、聚焦，不要宽泛、模糊。
  - 其他字段略
- 描述信息，技能的详细说明，是一段md格式的文本，这里记录的就是技能对应的“业务知识”。类似subAgent的定义，也类似提示词工程化后的角色描述。
  - 比如 mcp builder 的正文主体就是，MCP Server 的开发指南，包括概述，流程。流程里包括深度搜索和规划、实现、Review、评估，和里面的具体操作细节(详细内容自行搜索查看)
- 引用内容，SKILL.md 里面可以引用外部文件，**外部脚本(python语言，重点)** ，只要将对应文件和SKILL.md放到同一个目录下即可。
  - Claude 在运行脚本的时候会开启沙箱模式，确保安全。
定义技能后，不需要特别说明用哪个技能。通过自然语言理解，Claude Code 会自动识别和启用适合的技能来完成用户交付的任务。

### 真正意义！
介绍完基础定义和运行方式，想通过系列对比提问来认识 Agent Skill 的真正意义！
- 问：skill 和 subagent 有什么区别？
- 答：这两者我理解是升级替换关系。
  - 相似点，都是通过meta信息和提示词定义，都是容器自行识别、加载和运行。
  - 不同点，skill可以在Claude的多套产品中使用，subagent只是在 claude code 中使用。skill 可以运行预定义代码，subagent 不能
- 问：skill 和 plugin 有什么区别？
- 答：这两者是不同维度的东西。
  - skill是扩展ClaudeCode能力的一种方式(其他方式还有hooks、subAgents等)，
  - plugin是ClaudeCode所有扩展方式的定义能力，通过安装 plugin ，可以一次完成 plugin 定义的所有扩展能力。plugin有提供官方的市场可以对外发布，不使用 plugin 的话，所有的扩展能力都要传输文件，或者复制粘贴内容才能在团队内或互联网上流通。
- 问：skill 和 slash command (斜杠命令) 有什么区别？
- 答：这两者一个是系统主动触发，一个是用户主动触发
  - slash command 是要用户主动触发的，触发时机是在用户给ClaudeCode发信息之前
  - skill是ClaudeCode主动识别和触发的，过程不受用户控制，触发时机是在用户给ClaudeCode发信息之后。
- 问：skill 和 MCP 的区别？
- 答：这两者是本地和远程的区别。
  - skill 是完全本地运行的，运行环境受ClaudeCode控制，意味着安全和可控。即使运行py脚本，也是沙箱环境运行，安全系数高。
  - MCP 是跨进程运行的，就像RPC调用。优点是可快速集成已有的外部能力，缺点是性能(多一次网络调用，本地MCP无此影响)，有安全风险(被劫持，调到李鬼服务，甚至服务端运行的是黑盒，不知道实际运行情况)。
  - 不要忘记了，MCP协议本身就是 Anthropic 定义的，我大胆预感，**skill 和 MCP 未来还要进化和融合**。
- 问：skill 和 prompt engineering 的区别？
- 答：官方说明了，skil最大的价值之一，就是对 prompt 的复用，如果你发现使用 ClaudeCode 的时候，有重复对它讲的话，就应该用 skill 给他固化下来。(我重复最多的是 你给我认真想想，改不好拔你电源了！ 哈哈)。prompt engineering 是对 prompt 的改进，两者不冲突。
- 问：skill 和 tool use 的区别？
- 答：**重磅**，这俩本来没有交集，但我发现，skill 是 Claude 给普通用户的一次平权。
  - 最开始 LLM 用工具依赖 function call，只有模型训练人员知道。Agent开发者都不知道，要靠猜。所以当时说 ClaudeCode 用自家 sonnet 模型，效果就是要把外面的人好(对，就是说cursor要靠猜)
  - 然后到 Agent开发时代，开发框架提供了 ToolUse 的能力，只有开发者知道，使用 Agent 的用户是不知道的，用户也没法植入自定义的工具。
    - MCP也是在这个时代出现的，通常被封装成一个 Tool 在 Agent 中被调用。
  - 最后 skill 登场了！用户终于可以给Agent自定义工具了，Agent终于有了开放工具的自由度。
    - skill会不会发展成第二个MCP？ 可能事实不会，但各家都会跟进自定义工具这个事情。

### 总结
- Skill 是 Agent 开放性的重大突破，肯定会成为未来Agent的标配。
- 如果你或你的团队有重复对 Agent 说的 prompt，把它封装成 Skill 复用起来。

我会继续深度体验和定义自己的Skill，后续分享感受和实践。
