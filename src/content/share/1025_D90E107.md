## 1、50个VibeCoding技巧(8) Claude Agent Skills 解析
经过尝试官方的和构建自己的skills，我更加相信 skills 是 Agent 的范式革命。
官方文档描述 skill 的使用场景是 减少重复的 prompt 或者在团队间共享 知识、工具。减少重复性的话，subagent也能做到，共享的话，现在是依靠plugin来实现。我判断它更重要的作用是在
- 增加Agent的工具开放性，曾经Agent的工具只有开发者能预先定义，现在通过Skill可以让使用者把自己需要的工具放置进去。
- 提高Agent的确定性，在Agent流程内部，把可固化的内容用skill固定下来(最终执行预置的确定的py)。
所以skill应该定义那些最终可以用代码确切执行的功能(给Agent节省Token，并且减墒)

ClaudeCode一直在持续演进自己的开放性和复制便利性。
- 开放性，现在有 hooks，subAgent，skills，outputStyle(没激起什么水花),command 可以扩展
- 复制便利性，因为ClaudeCode都是基于md 或 json的，天生就方便复制，但因为可扩展点太多了，所以现在都统一通过收口到 plugin 来一键复制。甚至给 plugin 做了 marketplace 。

基于 skill 的Agent工具开放模式会快速被其他Agent采用，形式事实的产品标准，但是不会再有一次MCP了(MCP是蛮荒期的偷家行为，所有Agent最后都是不情不愿才支持的，google还推了一个A2A协议来对抗也没成功，活像曾经的大公司抱团推协议但抵挡不住开源方案的猛烈攻击，最后只能消亡掉，能否见证一次历史)。

ClaudeCode 的名字误导了大家，它一直是一个通用Agent，加上skill后，就更是了，超级生产力工具。我几乎所有事情都在 VSCode + ClaudeCode 的组合里完成。
