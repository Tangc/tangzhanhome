## 1、VibeCoding技巧(9) Cursor2.0 发布，VibeCoding体验无比丝滑？
Cursor发布了2.0，离我上一次打开它，好像已经过去了将近3个月。看看他有什么变化
### 变化1，发布自有编程模型 composor 1
Cursor作为一个三方CodeAgent的一大痛点是没有自己的模型，调用模型的成本高昂，这是相比一方模型厂商的CodeAgent的一大劣势。比如ClaudeCode可以开包月套餐，Cursor就只能一再削减额度。
作为中间方，用路过的编程数据来训练自有的编程模型几乎就是摆脱卡脖子的必然选项了。
但罕见得，composor 1作为新发布的模型，竟然没有屠榜，而只是强于sonnet4而弱于sonnet4.5和ChatGPT5，主打的是4倍速的高速执行。(想起采访刘小排的播客内容有提到这种高性价比的差异竞争方式)

正好接下来有个新项目，准备体验一下全 composor 编程能否完成，经验后续分享。

除了模型外，其他都是交互体验，体现了 Cursor 对 VibeCoding 的痛点理解，具体有
### 变化2，增加Agent模式
在Cursor的左上角增加一个 Agent 模式，切换后会变成类似 Lovable、Figma 之类的交互，把文件夹隐藏起来，左边栏显示当前启动的实例，右边显示和Agent的对话，和对话框。
有一种VibeCoding流派是完全托管型，自己是不看代码也不做任何手动修改的，或者本身开发者就是跨界者不会看源代码。根据我最近的体验，这类开发者也不在少数。
Agent模式下体验会更好，也给后续几个变化做了铺垫。

### 变化3，浏览器集成
Agent对话窗口里可以打开一个浏览器窗口，并且在此基础上还可以 
- 选中页面元素，直接加入到对话上下文
- 打开浏览器的控制台，devtools
- 截图加入上下文，截全屏加入上下文
对于网页开发者，非常的方便，指哪打哪，操作直观。

### 变化4，多Agent模式
Agent模式下，可以同时开多个Agent并行工作，并且可以选择 worktree 模式，底层应该就是 git worktree 实现的。

还有一些小变化还要慢慢探索，但都是交互上的体验。实话说，交互体验被追上是很快的，而且用vscode+插件的模式，完全可以搞出这么些功能来。对专业开发者来说可能还是没有打开Cursor的理由。
对纯VibeCoding的选手来说，有使用惯性，不会自己弄插件的话也还是会继续用Curosr的。这类里面用composor模型也完全够用，应该是 Cursor 接下来要主打的用户群了。
比 Lovable 等的用户更专业一点，比 ClaudeCode 的用户更浅一些，在这个层次里做到最好。
